Топология сети и цель работы

1 Общее описание

В этой лабораторной работе я настроил маршрутизатор Cisco для соединения двух отдельных сетей. Основные задачи включали:

  • Изменение hostname роутера на свою фамилию
  • Настройку интерфейсов с IP-адресами, где третий октет соответствует моему порядковому номеру в группе (28)
  • Настройку IP-адресов на четырех ПК
  • Проверку связности между сетями
Рис. 1 - Топология сети с роутером "Кабанен", соединяющим две сети

Подробные шаги настройки

2 Изменение отображаемого имени на топологии

Первым делом я изменил отображаемое имя роутера на схеме:

  1. Дважды кликнул по роутеру на схеме в Packet Tracer
  2. В открывшемся окне перешел на вкладку "Config"
  3. В поле "Display Name" ввел "Кабанен"
  4. Нажал "Apply" для сохранения изменений
Примечание: Это изменение влияет только на отображение имени на схеме, но не на фактический hostname устройства.

3 Настройка hostname через командную строку

Для изменения фактического hostname роутера я использовал CLI (Command Line Interface):

Router> enable Router# configure terminal Router(config)# hostname Кабанен Кабанен(config)#

После выполнения этих команд приглашение командной строки изменилось с "Router" на "Кабанен", что подтверждает успешное изменение hostname.

4 Настройка интерфейсов роутера

Роутер имеет два интерфейса, каждый из которых будет служить шлюзом для своей сети. Я использовал третий октет "28", соответствующий моему порядковому номеру.

Настройка интерфейса GigabitEthernet0/0 для первой сети:

Кабанен(config)# interface GigabitEthernet0/0 Кабанен(config-if)# ip address 192.168.28.1 255.255.255.0 Кабанен(config-if)# no shutdown Кабанен(config-if)# exit

Настройка интерфейса GigabitEthernet0/1 для второй сети:

Кабанен(config)# interface GigabitEthernet0/1 Кабанен(config-if)# ip address 172.16.28.1 255.255.255.0 Кабанен(config-if)# no shutdown Кабанен(config-if)# exit
Важно: Команда "no shutdown" активирует интерфейс. Без этой команды интерфейс останется в административно выключенном состоянии.

5 Настройка ПК

В топологии четыре ПК, по два в каждой сети. Я настроил их следующим образом:

Устройство IP-адрес Маска подсети Шлюз по умолчанию Подсеть
PC1 (левый) 192.168.28.10 255.255.255.0 192.168.28.1 Сеть 1
PC2 (левый) 192.168.28.20 255.255.255.0 192.168.28.1 Сеть 1
PC3 (правый) 172.16.28.10 255.255.255.0 172.16.28.1 Сеть 2
PC4 (правый) 172.16.28.20 255.255.255.0 172.16.28.1 Сеть 2
Ключевой момент: На всех ПК обязательно нужно заполнить поле "Gateway" (шлюз) адресом соответствующего интерфейса роутера. Без этого ПК не смогут отправлять пакеты в другую сеть.

Проверка работоспособности

6 Проверка состояния интерфейсов

После настройки я проверил состояние интерфейсов роутера:

Кабанен# show ip interface brief

Эта команда выводит таблицу со статусами всех интерфейсов. Оба интерфейса должны иметь статус "up" и протокол "up".

7 Проверка связности командой ping

Для проверки правильности настройки я выполнил команды ping с роутера:

Кабанен# ping 192.168.28.10 Кабанен# ping 172.16.28.10

Обе команды должны показать успешные ответы (процент успешных пинг-запросов должен быть 100%).

Рис. 2 - Проверка связности между ПК из разных сетей

Также я проверил связность между ПК из разных сетей (например, с PC1 на PC3), что подтвердило правильную работу маршрутизации.

8 Просмотр конфигурации и сохранение

Для просмотра текущей конфигурации можно использовать команду:

Кабанен# show running-config
Рис. 3 - Конфигурация роутера "Кабанен"

После успешной проверки всех настроек я сохранил конфигурацию в энергонезависимую память:

Кабанен# copy running-config startup-config Кабанен# write memory

Файл проекта был сохранен как Кабанен_роутер_4ПК_28.pkt, где "28" соответствует третьему октету всех IP-адресов.
